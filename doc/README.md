# Программа курса

## Введение для преподавателей

Цель данного курса: пропедевтика программирования.

Программирование - решение поставленных задач с помощью компьютера.

Необходимо научить писать законченные программы, а не просто изучение синтаксиса языка и нескольких библиотек.
Будем подходить к вопросу комплексно: оценка задачи, разбиение на подзадачи, структурирование программы. 

К вопросу обучения программированию можно подходить с двух сторон:
1. Решения практических задач любым удобным способом
2. Изучение многих деталей реализации языков и компьютеров, чтобы понимать как все работает и решать задачи эффективно.

Важно правильно выбрать инструмент. От этого зависит удобство в работе.
Думаю многие сталкивались с проблемой, что неудобные или неправильные инструменты осложняли выполнения работ.

Языков программирования много т.к. решаемых ими задач очень много.
Выберем тот, инструмент который подходит для обучения. 
Python выбран как очень лаконичный язык который прячет много деталей, чтобы не отвлекаться на технические детали. 

Естественно, такой подход чреват большими пробелами в знаниях, поэтому надо дать понять, чтобы освоить программирования по-настоящему одного Питона будет мало. 

Python лишь средство для достижения цели.

Язык Python достаточно коварен, на первый взгляд он подкупает своей простотой, но внутри кроется много
неочевидных деталей. Новичку они непонятны поэтому рано или поздно можно встретиться с непонятной ошибкой.

Но если есть цель по настоящему изучить программирования необходимо будет изучить и языки более низкого уровня(C, ассемблер).

>Метод - осознание формы внутренного самодвижения содержания изучаемого предмета.

>Цель - решить практические задачи

#### Введение

За тысячи лет своей истории  человек создал множество инструментов, чтобы приспособить окружающий
мир для своих целей.
Это вообще отличительная черта человека - создавать инструменты для преобразования мира.

Одним из таких инструментов является компьютер.
Идея компьютера очень интересна. Создаем устройство, которое может выполнять несколько команд.
С помощью набора этих команд можно создавать бесконечное число состояний.

Такой набор команд называется программа.

Программы пишутся людьми и для людей. Поэтому когда создаются программы надо в первую очередь
думать о тех, кто будет эту программу читать и применять.

Как разговаривать невозможно без человеческого языка, таки и программировать без компьютерного языка невозможно.

Для наших задач возьмем язык программирования Python. Он является одним из языков используемых в ЕГЭ.
Язык имеющий много достоинств и, конечно, множество недостатков (а где вы видели что-то без недостатков?).
Но для нашего курса он прекрасно подойдет.

Программа - это идея т.е. мысль отвечающая на вопрос "Что делать?"
Идея она не материальна (привести пример с яблоками).
В то же время не может существовать без материи (носителя информации).
 
Цель всего курса: заглянуть в мир создания программного обеспечения.
Формой курса будет разработка миниигру. Т.е. воплотить идею игры в виде работающей программы.

Хоть мы и будем делать игры, полученные навыки пригодятся для решения уже серьезных задач. Потомучто основные
принципы не меняются для любой программы.

Но эта цель для вас достаточно сложная и, думаю, многие сейчас не знают как ее решить.

Начало - неразвитый результат, результат - развернутое начало.
Т.е. в начале работы надо представлять, что будет результатом работы.
И когда все будет готово - это будет представление того, что мы задумывали в начале.

Чтобы решить сложную задачу есть такой популярный прием: разбить нее на ряд более простых.

Поэтому на каждом занятии будем ставить небольшие цели, позволяющие нам приблизить решения нашей главной цели.

#### Описание структуры програмы

Код 2 типов: пишем сейчас и написан ранее.
Программа состоит из комманд.

Код, написаный ранее находится в пакетах.
Что такое пакет?
Пакет может находится в пакете.
Если мы достаем что-то из пакета, то делаем это через точку.

Т.е. чтобы  с помощью компьютера решить задачу надо давать ему команды.

Мин 1 штука

Надо будет научиться как давать компьютеру команды, чтобы добиться желаемого результата.

Один вопрос: как узнать какие команды вводить?
Всегда есть автор, который эти команды придумывает. 
Или посмотреть в документации или спросить преподавателя.

Немного правил о том как давать имя команде.
Ограничения языка: без пробелов, вместо пробелов подчеркивание.

Объяснить про комментарии.

В ходе объяснения имена команд и переменных будут на английском.
Но если вы пишете код сами, то можно использовать русские имена.
Т.е. будет еще небольшой урок английского.

#### Простые циклы

Итак, наша первая цель: нарисовать оси.

Итак, цель поставлена, надо что-то сделать.

Все развивается от целого к целому, поэтому каждая наша команда будет
выполнять определенное законченное действие.

Как будет называться команда для этого действия?

`draw_axes()`

Скобочки нужны тобы показать что эту команду надо выполнить.

Мы с вами написали первую программу. Ее можно сохранить.

Но программы нужны чтобы из выполнять, поэтому запустим ее.
Но мы видим ошибку:
```
"NameError: name 'draw_axes' is not defined"
```

Здесь написано что компьютер не знает такого имени (т.е. такой команды)
что верно. Мы не объяснили компьютеру как надо эти оси рисовать.
А компьтеру нужна точность. Он выполняет только то, что вы в него ввели.

Итак, необходимо уточнить как нам рисовать оси.
Для этого нам понадобится команда def. Это команда и еще
несколько других встроены в язык авторами и компьютер про нее все знает.
Она нам поможет описать что мы хотим сделать.
```
def draw_axes():
	draw_x()
	draw_y()
	draw_z()
```
после команды def следует имя, далее в скобочках уточняющие параметры.
сейчас параметров никаких нет, поэтому ничего не пишем.
В конце ставится двоеточие как признак нового блока.

Такие команды называются функциями. Что такое функция - это некоторое
действие преобразование чегото в нечто совершенно иноое
например `y(x) = x*x`

Надо научиться отличать вызов программы от ее определения.
В начале мы написали вызов программы т.е. попросили компьютер выполнить.
После команды def мы уже конкретно указываем, что надо сделать.

Из каких шагов будет состоять наша функция.

Пишем реализацию функции `draw_x()`
Как нарисовать ось?
Расставить несколько кубиков в одну линию.
Здесь нам пригодиться готовая команда не зря же мы писали в начале import kubiki
используем вызов setBlock
на ее примере разбираем параметры функции
Добавляем константу для типа блока

Создается реализация из нескольких функций setBlock
Они заменяются на цикл.

Аналогично создаются функции для рисования остальных осей.

Если остается время можно попробовать написать обобщенную функцию.

Изучить что такое аргументы функции. Доработать функцию с аргументами.
Знакомство со словарями и кортежами.
Если остается время можно кортеж значений заменить на словарь.
